# Migration Guide: v0.9.2 → v0.9.3 Unified API

This guide helps you migrate from the v0.9.2 API to the new v0.9.3 unified API. The v0.9.3 release introduces significant improvements to the configuration system and engine initialization patterns.

## Breaking Changes Summary

### Removed Components

1. **`Presets` class** - The static `Presets` class has been completely removed
   - Old preset methods → Use `Engine(preset='default', mode='streaming')` or `get_preset('default')`
   - `Presets.throughput()` → Use `Engine(preset='default')` with `mode='batch'`
   - `Presets.validation()` → Use `Engine(preset='default')` with smaller parameters
   - `Presets.profiling()` → Use `Engine(preset='default', enable_profiling=True)`
   - `Presets.custom(**kwargs)` → Use `get_preset('default', **kwargs)` or `EngineConfig.from_preset()`

2. **Old Preset Names** - Previous preset names have been consolidated
   - New presets: `'default'`, `'iono'`, `'ionox'`
   - Focus on domain-specific configurations (ionosphere research)

### Changed Components

1. **`Engine` Constructor** - New initialization patterns
   - OLD: `Engine(config)` (single positional argument)
   - NEW: `Engine(preset=..., config=..., pipeline=..., **overrides)` (keyword arguments with three patterns)

2. **`EngineConfig`** - Unified configuration consolidating all parameters
   - Added pipeline parameters: `window`, `window_symmetry`, `window_norm`, `scale`, `output`
   - Added execution parameters: `mode`, `stream_count`, `pinned_buffer_count`
   - Removed separate config classes (all in one unified `EngineConfig`)

### New Components

1. **Preset Functions** - Replace `Presets` class with functions
   - `get_preset(name, **overrides)` - Get preset configuration
   - `list_presets()` - List available presets
   - `describe_preset(name)` - Get preset details
   - `compare_presets(names)` - Compare multiple presets

2. **`PipelineBuilder`** - Fluent interface for custom pipelines
   - Build custom processing pipelines with method chaining
   - Full control over stages, parameters, and configuration

3. **Expanded Enums** - More configuration options
   - `WindowSymmetry` - PERIODIC, SYMMETRIC
   - `WindowNorm` - UNITY, SQRT
   - `OutputMode` - MAGNITUDE, COMPLEX

## Migration Examples

### Example 1: Basic Engine Initialization

**Old (v0.9.2):**
```python
from ionosense_hpc import Engine
from ionosense_hpc.config import Presets

# Create engine with streaming mode
config = Presets.realtime()  # Old method
engine = Engine(config)
```

**New (v0.9.3):**
```python
from ionosense_hpc import Engine

# Pattern 1: Direct preset (recommended)
engine = Engine(preset='default', mode='streaming')

# Pattern 2: Using get_preset
from ionosense_hpc.config import get_preset
config = get_preset('default')
config.mode = 'streaming'
engine = Engine(config=config)

# Pattern 3: Using from_preset factory
from ionosense_hpc import EngineConfig
config = EngineConfig.from_preset('default', mode='streaming')
engine = Engine(config=config)
```

### Example 2: Custom Configuration

**Old (v0.9.2):**
```python
from ionosense_hpc import Engine
from ionosense_hpc.config import Presets

# Create custom configuration
config = Presets.custom(
    nfft=4096,
    batch=8,
    overlap=0.75
)
engine = Engine(config)
```

**New (v0.9.3):**
```python
from ionosense_hpc import Engine

# Pattern 1: Quick overrides (recommended)
engine = Engine(preset='iono', nfft=4096, batch=8, overlap=0.75)

# Pattern 2: Using get_preset
from ionosense_hpc.config import get_preset
config = get_preset('iono', nfft=4096, batch=8, overlap=0.75)
engine = Engine(config=config)

# Pattern 3: Using from_preset factory
from ionosense_hpc import EngineConfig
config = EngineConfig.from_preset('iono', nfft=4096, batch=8, overlap=0.75)
engine = Engine(config=config)

# Pattern 4: Full custom configuration
from ionosense_hpc import EngineConfig
config = EngineConfig(
    nfft=4096,
    batch=8,
    overlap=0.75,
    window='blackman',
    mode='batch'
)
engine = Engine(config=config)
```

## Configuration Parameter Mapping

### Preset Mapping

| Old Preset (v0.9.2) | New Pattern (v0.9.3) | Notes |
|---------------------|----------------------|-------|
| Old streaming preset | `Engine(preset='default', mode='streaming')` | Default preset with streaming mode |
| `Presets.throughput()` | `Engine(preset='default', mode='batch')` | Default preset with batch mode (default) |
| `Presets.validation()` | `Engine(preset='default', batch=1)` | Default preset with minimal batch |
| `Presets.profiling()` | `Engine(preset='default', enable_profiling=True)` | Default preset with profiling enabled |
| `Presets.custom(**kwargs)` | `get_preset('default', **kwargs)` | Use get_preset or from_preset |
| N/A | `Engine(preset='iono')` | **NEW** Ionosphere research preset |
| N/A | `Engine(preset='ionox')` | **NEW** Extreme ionosphere preset |

### EngineConfig Parameters

All previous `EngineConfig` parameters are preserved. New parameters added:

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `window` | `WindowType` | `HANN` | Window function type |
| `window_symmetry` | `WindowSymmetry` | `PERIODIC` | Window symmetry mode |
| `window_norm` | `WindowNorm` | `UNITY` | Window normalization |
| `scale` | `ScalePolicy` | `ONE_OVER_N` | FFT scaling policy |
| `output` | `OutputMode` | `MAGNITUDE` | Output format |
| `mode` | `ExecutionMode` | `BATCH` | Execution strategy |
| `stream_count` | `int` | `3` | Number of CUDA streams |
| `pinned_buffer_count` | `int` | `2` | Number of pinned buffers |

## New Capabilities in v0.9.3

### 1. Preset Functions

Replace the static `Presets` class with more flexible functions:

```python
from ionosense_hpc.config import get_preset, list_presets, describe_preset, compare_presets

# Get available presets
presets = list_presets()  # ['default', 'iono', 'ionox']

# Get preset details
info = describe_preset('iono')
print(info['description'])
print(info['parameters'])

# Compare presets
comparison = compare_presets(['default', 'iono', 'ionox'])
```

### 2. PipelineBuilder for Custom Pipelines

Build custom processing pipelines with fluent interface:

```python
from ionosense_hpc import Engine, PipelineBuilder

pipeline = (
    PipelineBuilder()
    .add_window('blackman', symmetry='periodic', norm='unity')
    .add_fft(scale='1/N')
    .add_magnitude()
    .configure(nfft=4096, batch=8, overlap=0.75)
    .build()
)

engine = Engine(pipeline=pipeline)
```

### 3. Three Initialization Patterns

Choose the pattern that best fits your use case:

```python
from ionosense_hpc import Engine

# Pattern 1: Preset-based (simplest, recommended for common cases)
engine = Engine(preset='iono', nfft=8192)

# Pattern 2: Config-based (full control)
from ionosense_hpc import EngineConfig
config = EngineConfig(nfft=4096, batch=8, overlap=0.75, window='blackman')
engine = Engine(config=config)

# Pattern 3: Pipeline-based (advanced custom pipelines)
from ionosense_hpc import PipelineBuilder
pipeline = PipelineBuilder().add_window(...).add_fft(...).build()
engine = Engine(pipeline=pipeline)
```

### 4. Execution Modes

New `ExecutionMode` enum for explicit execution strategy:

```python
from ionosense_hpc import Engine, ExecutionMode

# Batch mode (default) - maximum throughput
engine = Engine(preset='iono', mode=ExecutionMode.BATCH)

# Streaming mode - low-latency streaming with ring buffer
engine = Engine(preset='iono', mode=ExecutionMode.STREAMING)
```

### 5. Window Configuration

New window configuration options:

```python
from ionosense_hpc import Engine, WindowType, WindowSymmetry, WindowNorm

engine = Engine(
    preset='iono',
    window=WindowType.BLACKMAN,
    window_symmetry=WindowSymmetry.PERIODIC,  # For FFT processing
    window_norm=WindowNorm.UNITY              # Unity power gain
)

# Or use string values (automatically converted)
engine = Engine(
    preset='iono',
    window='blackman',
    window_symmetry='periodic',
    window_norm='unity'
)
```

## Common Migration Issues

### Issue 1: Import Errors

**Problem:**
```python
ImportError: cannot import name 'Presets' from 'ionosense_hpc.config'
```

**Solution:**
```python
# OLD
from ionosense_hpc.config import Presets

# NEW
from ionosense_hpc.config import get_preset, list_presets
# or
from ionosense_hpc import Engine  # Use Engine directly with preset parameter
```

### Issue 2: Positional Argument Errors

**Problem:**
```python
TypeError: Engine() takes 0 positional arguments but 1 was given
```

**Solution:**
```python
# OLD
engine = Engine(config)

# NEW
engine = Engine(config=config)
# or
engine = Engine(preset='default', **overrides)
```

### Issue 3: Missing Preset Names

**Problem:**
```python
ValueError: Unknown preset: 'realtime'
```

**Solution:**
```python
# OLD
engine = Engine(preset='realtime')

# NEW
engine = Engine(preset='default', mode='streaming')
```

### Issue 4: Configuration Attribute Errors

**Problem:**
```python
AttributeError: 'EngineConfig' object has no attribute 'window'
```

**Solution:** You're using an old `EngineConfig` object. Recreate it with the new unified config:

```python
# NEW
from ionosense_hpc import EngineConfig
config = EngineConfig(
    nfft=4096,
    batch=8,
    overlap=0.75,
    window='blackman',  # Now supported
    mode='batch'        # Now supported
)
```
